## **2. Autonomous Mobile Robot Dispatching System (AMRDS)** 

In modern smart factories and automated fulfillment centers, **Autonomous Mobile Robots (AMRs)** serve as the backbone of materials handling, moving components dynamically between work cells. Mathematically, the factory floor grid is modeled as a **Graph** : 

- **Nodes:** Represent physical stations/cells (e.g., Charging Docks, Assembly Cells, Inventory Racks). 

• **Edges:** Represent the physical tracks or pathways connecting those stations. To manage a team of AMRs efficiently, the central dispatching program must address three structural challenges: 

1. **Dynamic State Management:** Each robot undergoes a complex lifecyle (Idle, Moving, Charging, Faulted). Dispatch behaviors must dynamically morph based on the robot's active state. 

2. **Hardware Extensibility:** Smart factories continuously integrate new robot generations (e.g., Heavy Load Pallet Movers, Inventory Scanners). The program must support new hardware models without mutating existing dispatch algorithms. 

3. **Generic Pathfinding Costs:** Routing optimization matrices vary by operational context— calculating by _Geometric Distance (meters)_ , _Transit Time (seconds)_ , or _Battery Consumable Overhead (Watt-hours)_ . The path-finding module must be entirely decoupled from explicit edge weight primitives. 

The execution lifecycle runs entirely automated via an end-to-end file pipeline. The core engine parses an infrastructure layout file, processes the queue, and writes a detailed dispatching ledger to an output text document: 

[input_file.txt] ──> [Core AMRDS Engine] ──> [output_file.txt] (Map, Fleet, Command Queue) (Factory, State, Template) (Dispatch Ledger & Optimal Paths)
